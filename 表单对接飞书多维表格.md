# 官网表单 → 飞书多维表格

表单提交 → 后端接口 `/api/lead` → 用飞书自建应用 token 写一条记录进多维表格。

所有 token、表 ID 都走**环境变量**，本文档不含任何真实值，可安全提交。

---

## 字段映射

| 表单 payload | 多维表格字段 | 备注 |
|---|---|---|
| `name` | 姓名 | 必填 |
| `company` | 公司 | 必填 |
| `phone` | 手机号 | 必填 |
| `industry` | 行业 | |
| `message` | 留言 | |
| `source` | 来源 | 需在表里加这列；否则丢弃 |
| `page` | 页面 | 需在表里加这列；否则丢弃 |
| —（自动） | 录入时间 | 飞书自动填 |

---

## 一、环境变量（配在服务器，别进 git）

```bash
# /opt/www/juzibot/.env
FEISHU_APP_ID=             # 自建应用 app_id，cli_ 开头
FEISHU_APP_SECRET=         # 自建应用密钥，保密
FEISHU_BASE_APP_TOKEN=     # 表链接 /base/<这段>
FEISHU_BASE_TABLE_ID=      # 表链接 ?table=<这段>
```

> 表链接形如 `https://xxx.feishu.cn/base/{APP_TOKEN}?table={TABLE_ID}`，后两个值从这里取。
> **先在 `.gitignore` 加一行 `.env`。**

## 二、建应用 + 授权这张表

1. [open.feishu.cn](https://open.feishu.cn) 建**企业自建应用** → 拿 `app_id` + `app_secret`。
2. 权限管理开 `bitable:record:create` → 创建版本并**发布上线**。
3. 打开多维表格 → 协作者 → 加这个应用 →「**可编辑**」。
   （只开权限不把表加给应用，照样写不进去。）

## 三、后端接口（FastAPI，全读环境变量）

服务器已有 `python3`。`pip install fastapi uvicorn httpx`。

```python
# server/lead_api.py
import os, time, httpx
from fastapi import FastAPI, Request

APP_ID     = os.environ["FEISHU_APP_ID"]
APP_SECRET = os.environ["FEISHU_APP_SECRET"]
BASE       = os.environ["FEISHU_BASE_APP_TOKEN"]
TABLE      = os.environ["FEISHU_BASE_TABLE_ID"]
URL        = "https://open.feishu.cn/open-apis"

app = FastAPI()
_tok = {"v": None, "exp": 0}

async def token(client):
    if _tok["v"] and time.time() < _tok["exp"] - 120:
        return _tok["v"]
    d = (await client.post(f"{URL}/auth/v3/tenant_access_token/internal",
         json={"app_id": APP_ID, "app_secret": APP_SECRET})).json()
    _tok["v"], _tok["exp"] = d["tenant_access_token"], time.time() + d["expire"]
    return _tok["v"]

@app.post("/api/lead")
async def lead(req: Request):
    p = await req.json()
    if not (p.get("name") and p.get("company") and p.get("phone")):
        return {"ok": False, "error": "缺少必填字段"}
    fields = {
        "姓名": p.get("name", ""), "公司": p.get("company", ""),
        "手机号": p.get("phone", ""), "行业": p.get("industry", ""),
        "留言": p.get("message", ""),
        # 表里加了「来源」「页面」列就解开：
        # "来源": p.get("source", ""), "页面": p.get("page", ""),
    }
    async with httpx.AsyncClient(timeout=10) as c:
        d = (await c.post(f"{URL}/bitable/v1/apps/{BASE}/tables/{TABLE}/records",
             headers={"Authorization": f"Bearer {await token(c)}"},
             json={"fields": fields})).json()
    return {"ok": d.get("code") == 0, "msg": d.get("msg")}
```

启动（systemd 或 tmux 常驻）：

```bash
cd /opt/www/juzibot && set -a; source .env; set +a
uvicorn server.lead_api:app --host 127.0.0.1 --port 8910
```

nginx 反代加进现有 server 块：

```nginx
location /api/lead {
    proxy_pass http://127.0.0.1:8910/api/lead;
}
```

## 四、前端指向接口

在 `index.html` 的 `<script>` 顶部、`build_pages.py` 页面模板的脚本顶部各加一行：

```js
window.JUZI_LEAD_ENDPOINT = "/api/lead";
```

改 `build_pages.py` 后跑 `python3 build_pages.py` 重建子页（部署脚本会自动重建）。

## 本地手动写入 / 补录（lark-cli，可选）

用个人登录态临时写，**不要用在常驻服务器**（token 几天过期）。值从 `.env` 取，不写进命令历史的明文：

```bash
set -a; source .env; set +a
lark-cli api POST \
  "/open-apis/bitable/v1/apps/$FEISHU_BASE_APP_TOKEN/tables/$FEISHU_BASE_TABLE_ID/records" \
  --data '{"fields":{"姓名":"测试","公司":"句子互动","手机号":"13800000000"}}' \
  --jq '{code,msg}'
```

> 已验证这张表用 lark-cli 写入 → 回读 → 删除全部成功（`code: 0`）。

## 安全 checklist

- [ ] `.gitignore` 加 `.env`
- [ ] `app_secret` 只在服务器环境变量，不进 git / 文档 / 聊天
- [ ] 接口加限频 + 手机号格式校验，防灌垃圾线索
- [ ] 应用只开 `bitable:record:create`，表只授权给该应用
