# 官网表单 → 飞书多维表格 对接说明

> 本文档说明：官网「联系我们」表单如何把线索写进飞书多维表格。
> 包含三部分：① 现状结论 ② 用 lark-cli 本地验证写入（已跑通）③ 线上生产方案（服务器自建应用 token）。

---

## 一、现状结论（截至本次检查）

官网表单**目前不会把数据写进任何地方**。

所有 18 个页面 + `build_pages.py` 模板里的提交逻辑都是这段：

```js
var ep = window.JUZI_LEAD_ENDPOINT;
if (!ep) { done(); return; }   // ep 全站从未定义 → 永远走这行 → 直接弹成功提示
fetch(ep, { method:'POST', ... })   // 永远不执行
```

- `window.JUZI_LEAD_ENDPOINT` 这个变量在全站任何文件里都没有定义。
- 用户点「提交」后只看到成功提示，**数据被丢弃**。
- 仓库里没有后端 / Worker / 飞书集成代码。

要真正写进多维表格，需要补两样东西：

1. 一个**后端接口**（接收表单 POST，再写进 Bitable）；
2. 把这个接口地址赋值给 `window.JUZI_LEAD_ENDPOINT`。

官网是**自建服务器**部署（`.github/workflows/deploy.yml`：SSH 拉 `main` 到 `/opt/www/juzibot`），所以可以在同一台机上加这个后端接口。

## 二、目标多维表格

| 项 | 值 |
|---|---|
| 多维表格链接 | https://juzihudong.feishu.cn/base/JPfJboxn2aIIgPsNXXLcDF8dnth?table=tbljLQoxTMeXwI9K |
| `app_token` | `JPfJboxn2aIIgPsNXXLcDF8dnth` |
| `table_id` | `tbljLQoxTMeXwI9K` |

> `app_token` / `table_id` 不是密钥（在分享链接里就有），可以放文档。**真正的密钥是自建应用的 `app_secret`，绝不能进文档 / git / 聊天。**

### 表单字段 → 表格字段映射

| 表单字段（payload key） | 多维表格字段 | 类型 | 说明 |
|---|---|---|---|
| `name` | 姓名 | 文本 | 必填 |
| `company` | 公司 | 文本 | 必填 |
| `phone` | 手机号 | 文本 | 必填 |
| `industry` | 行业 | 文本 | 下拉选项文本 |
| `message` | 留言 | 文本 | 选填 |
| —（自动） | 录入时间 | 创建时间 | 飞书自动填，无需写 |
| `source` | **（表里暂无）** | — | 线索来自哪个按钮/弹窗，建议加列 |
| `page` | **（表里暂无）** | — | 线索来自哪个页面，建议加列 |

> 建议在表里加两列「来源」「页面」（文本），把 `source` / `page` 也写进去，方便线索归因。加列后在后端 `fields` 里补上对应键即可。

## 三、用 lark-cli 本地验证写入（已跑通）

lark-cli 用的是**你个人登录的 token**（`李佳芮`），适合本地手动验证、补录数据，**不适合常驻线上服务器**（token 每几天过期）。

前置：`lark-cli auth status` 确认已登录；过期则 `lark-cli auth login --recommend`。

### 1. 读字段（确认字段名/类型）

```bash
APP=JPfJboxn2aIIgPsNXXLcDF8dnth
TBL=tbljLQoxTMeXwI9K
lark-cli api GET "/open-apis/bitable/v1/apps/$APP/tables/$TBL/fields" \
  --params '{"page_size":100}' --jq '.data.items[] | {field_name, type, ui_type}'
```

### 2. 写一条记录

```bash
lark-cli api POST "/open-apis/bitable/v1/apps/$APP/tables/$TBL/records" \
  --data '{"fields":{"姓名":"测试","公司":"句子互动","手机号":"13800000000","行业":"在线教育","留言":"测试记录"}}' \
  --jq '{code,msg,record_id:.data.record.record_id}'
# → {"code":0,"msg":"success","record_id":"recxxxx"}
```

### 3. 回读 / 删除

```bash
RID=recxxxx
lark-cli api GET    "/open-apis/bitable/v1/apps/$APP/tables/$TBL/records/$RID" --jq '.data.record.fields'
lark-cli api DELETE "/open-apis/bitable/v1/apps/$APP/tables/$TBL/records/$RID" --jq '{code,msg,deleted:.data.deleted}'
```

> 本次已用上述命令完成 写入 → 回读 → 删除，全部 `code: 0`，**确认 lark-cli 可成功写入这张表**。

## 四、线上生产方案：服务器自建应用 token（推荐）

线上不用个人 token，改用**飞书企业自建应用**的 `app_id` + `app_secret`。服务器用这俩在运行时换取 `tenant_access_token`（有效期 2 小时，自动续），再写记录。token 永不因个人登录过期。

### 第 1 步：建自建应用，拿到要给服务器的两个值

1. 打开 [飞书开放平台](https://open.feishu.cn) → 开发者后台 → 创建**企业自建应用**（名字如「官网线索写入」）。
2. 在「凭证与基础信息」里拿到两个值——**这就是要提供给服务器的全部凭证**：

   | 要给服务器的值 | 来源 | 性质 |
   |---|---|---|
   | `app_id`（形如 `cli_xxxxx`） | 凭证与基础信息 | 可公开 |
   | `app_secret` | 凭证与基础信息 | **密钥，保密** |

3. 「权限管理」里开通权限：
   - `bitable:record:create`（写记录，必需）
   - 如需读/改：`bitable:record:read`、`bitable:record:update`
4. 创建版本并**发布上线**（自建应用要发布后权限才生效）。

### 第 2 步：把这张表授权给应用

光有权限还不够，应用要能访问**这张具体的表**：

- 打开多维表格 → 右上角「...」或「协作者」→ 添加协作者 → 搜应用名 → 给「**可编辑**」。
- （或在应用「数据权限」里把该 Base 加进可见范围。）

### 第 3 步：把凭证配到服务器（不进 git）

在服务器 `/opt/www/juzibot` 用环境变量，**不要写进代码或提交**：

```bash
# /opt/www/juzibot/.env   （确认 .env 已在 .gitignore；本仓库 .gitignore 已忽略 *.log 等，需补 .env）
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_BASE_APP_TOKEN=JPfJboxn2aIIgPsNXXLcDF8dnth
FEISHU_BASE_TABLE_ID=tbljLQoxTMeXwI9K
```

> **总结「给服务器提供哪些 token」**：只给 `app_id` + `app_secret`（再加表的 `app_token` + `table_id` 两个非密钥定位值）。服务器自己换 `tenant_access_token`，不需要、也不要把个人 lark-cli token 放上去。

### 第 4 步：后端接口（参考实现，Python / FastAPI）

服务器已有 `python3`。装 `fastapi uvicorn httpx`，跑一个小服务，nginx 反代到 `/api/lead`。

```python
# server/lead_api.py
import os, time, httpx
from fastapi import FastAPI, Request

APP_ID     = os.environ["FEISHU_APP_ID"]
APP_SECRET = os.environ["FEISHU_APP_SECRET"]
BASE       = os.environ["FEISHU_BASE_APP_TOKEN"]
TABLE      = os.environ["FEISHU_BASE_TABLE_ID"]
BASE_URL   = "https://open.feishu.cn/open-apis"

app = FastAPI()
_token = {"value": None, "exp": 0}

async def tenant_token(client):
    if _token["value"] and time.time() < _token["exp"] - 120:
        return _token["value"]
    r = await client.post(f"{BASE_URL}/auth/v3/tenant_access_token/internal",
                          json={"app_id": APP_ID, "app_secret": APP_SECRET})
    d = r.json()
    _token["value"] = d["tenant_access_token"]
    _token["exp"]   = time.time() + d["expire"]
    return _token["value"]

@app.post("/api/lead")
async def lead(req: Request):
    p = await req.json()
    if not (p.get("name") and p.get("company") and p.get("phone")):
        return {"ok": False, "error": "缺少必填字段"}
    fields = {
        "姓名":   p.get("name", ""),
        "公司":   p.get("company", ""),
        "手机号": p.get("phone", ""),
        "行业":   p.get("industry", ""),
        "留言":   p.get("message", ""),
        # 若已在表里加「来源」「页面」列，解开下面两行：
        # "来源": p.get("source", ""),
        # "页面": p.get("page", ""),
    }
    async with httpx.AsyncClient(timeout=10) as client:
        tok = await tenant_token(client)
        r = await client.post(
            f"{BASE_URL}/bitable/v1/apps/{BASE}/tables/{TABLE}/records",
            headers={"Authorization": f"Bearer {tok}"},
            json={"fields": fields},
        )
    d = r.json()
    return {"ok": d.get("code") == 0, "code": d.get("code"), "msg": d.get("msg")}
```

启动（用 systemd 或 tmux 常驻）：

```bash
cd /opt/www/juzibot
set -a; source .env; set +a
uvicorn server.lead_api:app --host 127.0.0.1 --port 8910
```

nginx 反代（加到现有 server 块）：

```nginx
location /api/lead {
    proxy_pass http://127.0.0.1:8910/api/lead;
    proxy_set_header Content-Type application/json;
}
```

### 第 5 步：前端指向接口

给页面定义 `JUZI_LEAD_ENDPOINT`。最简单：在 `build_pages.py` 的页面模板和 `index.html` 的 `<script>` 顶部加一行：

```js
window.JUZI_LEAD_ENDPOINT = "/api/lead";
```

加上后，表单提交就会真正 POST 到后端 → 写进多维表格。改完 `index.html` 直接生效；改 `build_pages.py` 后 `python3 build_pages.py` 重新生成子页（部署脚本会自动重建）。

## 五、安全 checklist

- [ ] `app_secret` 只存服务器环境变量，**不进 git / 不进本文档 / 不发聊天**。
- [ ] 服务器 `.env` 加进 `.gitignore`（当前仓库 `.gitignore` 未忽略 `.env`，需补一行 `.env`）。
- [ ] 后端接口加基本防刷（限频 / 简单校验手机号格式），否则会被灌垃圾线索。
- [ ] 自建应用权限按最小需要开（只开 `bitable:record:create`）。
- [ ] 这张表只授权给该应用「可编辑」，不要给多余协作者。
