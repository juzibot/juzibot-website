2025 年 LLM 年度回顾

2025年12月19日

2025 年是 LLM 领域进展强劲且大事频发的一年。以下是我个人认为值得注意且略感意外的"范式转变"——那些改变了格局、在概念上令我印象深刻的事物。

1. 基于可验证奖励的强化学习 (RLVR)

2025 年初，所有实验室的 LLM 生产技术栈大致如下：

  1. 预训练 (约 2020 年的 GPT-2/3)
  2. 监督微调 (约 2022 年的 InstructGPT) 以及
  3. 基于人类反馈的强化学习 (约 2022 年的 RLHF)

这曾是训练生产级 LLM 一段时间内稳定且经过验证的配方。2025 年，基于可验证奖励的强化学习 (RLVR) 作为事实上的新主要阶段加入其中。通过在多种环境中（例如数学/代码谜题）针对可自动验证的奖励训练 LLM，LLM 自发地发展出在人类看来类似"推理"的策略——它们学会将问题解决分解为中间计算步骤，并学会多种问题解决策略，通过反复推敲来弄清问题（参见 DeepSeek R1 论文中的示例）。在之前的范式中，这些策略很难实现，因为对于 LLM 来说，最优的推理轨迹和恢复路径并不明确——它必须通过针对奖励的优化来找到适合自己的方法。

与 SFT 和 RLHF 阶段（两者都是相对轻量/短暂的阶段，计算上只是微调）不同，RLVR 涉及针对客观（不可操纵）的奖励函数进行训练，这允许进行更长时间的优化。事实证明，运行 RLVR 提供了很高的能力/成本比，这吞噬了原本用于预训练的计算资源。因此，2025 年的大部分能力进展都由 LLM 实验室消化这一新阶段的潜力所定义，总体上我们看到大小相似的 LLM，但 RL 运行时间长得多。同样独特于这个新阶段的是，我们获得了一个全新的调节旋钮（以及相关的缩放定律），通过生成更长的推理轨迹和增加"思考时间"，来控制能力作为测试时计算量的函数。OpenAI o1 (2024 年末) 是 RLVR 模型的首次展示，但 o3 的发布 (2025 年初) 是显而易见的转折点，你可以直观地感受到其中的差异。

2. 幽灵 vs. 动物 / 参差不齐的智能

2025年是我（我想整个行业也是）首次开始以更直观的方式理解LLM智能的“形态”。我们不是在“进化/培育动物”，而是在“召唤幽灵”。LLM技术栈的方方面面都不同（神经网络架构、训练数据、训练算法，尤其是优化压力），因此我们在智能领域得到截然不同的实体也就不足为奇了，用看待动物的视角去理解它们并不合适。从监督信号的角度看，人类神经网络是为了在丛林中生存而优化的，而LLM神经网络则是为了模仿人类文本、在数学谜题中获取奖励、以及在LM Arena上获得人类点赞而优化的。随着可验证领域允许强化学习可验证奖励（RLVR），LLM在这些领域附近的能力会“飙升”，整体表现出有趣的锯齿状性能特征——它们同时是天才的博学者，也是困惑且认知受限的小学生，随时可能被越狱手段欺骗而泄露你的数据。

(人类智能：蓝色，AI智能：红色。我喜欢这个版本的梗图（抱歉我丢失了它在X上原始帖子的引用），因为它指出了人类智能也以自身不同的方式呈现锯齿状。)

与此相关的是我在2025年对基准测试普遍感到的漠然和失去信任。核心问题在于，基准测试几乎在构造上就是可验证的环境，因此立即容易受到RLVR及其通过合成数据生成的较弱形式的影响。在典型的基准测试刷分过程中，LLM实验室的团队不可避免地会构建与基准测试所占据的小块嵌入空间相邻的环境，并生长出锯齿状的能力来覆盖它们。在测试集上训练成了一种新的艺术形式。

3. Cursor / LLM应用的新层级

关于 Cursor，除了它今年的异军突起之外，我认为最值得注意的是，它令人信服地揭示了“LLM 应用”的一个新层级——人们开始谈论“X 领域的 Cursor”。正如我今年在 Y Combinator 演讲中所强调的（文字稿 和 视频），像 Cursor 这样的 LLM 应用，会针对特定垂直领域捆绑并编排 LLM 调用：

  1. 它们负责“上下文工程”
  2. 它们在底层编排多个 LLM 调用，将其串联成日益复杂的 DAG（有向无环图），并精心权衡性能与成本。
  3. 它们为“人在回路”提供特定于应用的图形用户界面
  4. 它们提供一个“自主性滑块”

2025 年，关于这个新应用层有多“厚实”的讨论不绝于耳。LLM 实验室会包揽所有应用，还是 LLM 应用仍有广阔天地？我个人推测，LLM 实验室将趋向于培养出能力全面的“大学毕业生”，而 LLM 应用则会通过提供私有数据、传感器、执行器和反馈循环，来组织、微调并真正驱动这些“毕业生”团队，使它们成为在特定垂直领域部署就绪的“专业人士”。

4. Claude Code / 驻留在你电脑上的 AI

Claude Code (CC) 是首个令人信服地展示 LLM Agent 形态的实例——它以循环调用的方式串联工具使用与推理过程，从而实现长时间的问题求解。此外，CC 的独特之处在于它运行在用户本地计算机上，能直接访问私有环境、数据和上下文。我认为 OpenAI 在这方面走了弯路，因为他们早期的 Codex/Agent 项目过度聚焦于通过 ChatGPT 调度容器的云端部署方案，而非简单的 localhost 本地化路径。虽然云端运行的智能体集群看似"AGI 终局形态"，但在当前能力参差不齐、发展渐进平缓的过渡阶段，让智能体直接在开发者计算机上运行显然更具现实意义。关键区别其实不在于"AI 运算"发生的位置（云端、本地或其他），而在于其他所有因素——那些早已就绪并启动的计算机系统、其安装环境、上下文数据、密钥信息、配置参数，以及低延迟的交互体验。Anthropic 准确把握了这种优先级，将 CC 打包成精巧的命令行工具，重新定义了 AI 的存在形态：它不再是你像访问 Google 那样需要登录的网站，而是栖居在你计算机中的"数字精灵"。这构成了人机交互的全新范式。

5. 氛围编程

2025年是AI跨越能力阈值的一年，从此只需通过英文描述就能构建各种令人惊叹的程序，甚至让人忘记代码本身的存在。有趣的是，我在这条灵感迸发的推文中创造了"vibe coding"这个术语，完全没料到它会产生如此深远的影响 :)。通过vibe coding，编程不再仅仅是训练有素的专业人士的专利，而是任何人都能参与的事情。从这个意义上说，它恰好印证了我在《权力归于人民：大语言模型如何颠覆技术扩散模式》一文中探讨的观点——与以往所有技术截然不同的是，普通民众从大语言模型中获得的益处远超过专业人士、企业和政府。但vibe coding不仅让普通人能够接触编程，也让受过训练的专业人士能够编写更多（通过vibe coding完成的）原本不会被开发的软件。在nanochat项目中，我用Rust语言vibe coded了自己定制的高效BPE分词器，而无需采用现有库或深入学习Rust。今年我还通过vibe coding快速构建了许多项目，作为我想实现的应用原型（例如menugen、llm-council、reader3、HN time capsule）。我甚至曾为了定位一个bug而vibe coded整个临时应用——既然代码突然变得免费、临时、可塑且用后即弃，何乐而不为呢？Vibe coding将重塑软件生态，并改变职业描述。

6. Nano banana / LLM GUI

Google Gemini Nano banana 是2025年最令人惊叹、最具颠覆性的模型之一。在我看来，LLM是下一个主要的计算范式，类似于1970年代、80年代的计算机。因此，我们将看到类似的根本性创新，其背后的原因也相似。我们将看到个人计算、微控制器（认知核心）、互联网（智能体网络）等事物的对应版本。特别是在用户界面体验方面，与LLM“对话”有点像1980年代向计算机控制台发出命令。文本是计算机（和LLM）的原始/首选数据表示形式，但它并非人类偏好的格式，尤其是在输入端。人们实际上不喜欢阅读文本——它既慢又费力。相反，人们喜欢以视觉和空间方式获取信息，这就是传统计算中发明图形用户界面（GUI）的原因。同样，LLM也应该以我们偏好的格式与我们交流——图像、信息图表、幻灯片、白板、动画/视频、网络应用等。当然，这方面早期和现在的版本包括表情符号和Markdown等，它们是通过标题、粗体、斜体、列表、表格等方式“装饰”和视觉化排版文本，使其更易于消费。但究竟谁来构建LLM的GUI呢？在这个世界观中，nano banana是这种可能性的第一个早期迹象。重要的是，它的一个显著特点不仅在于图像生成本身，还在于文本生成、图像生成和世界知识共同产生的联合能力，这些能力都纠缠在模型权重中。

太长不看版。2025年是LLM领域令人兴奋且略带惊喜的一年。LLM正作为一种新型智能涌现，它同时比我预期的要聪明得多，也比我预期的要笨拙得多。无论如何，它们极其有用，我认为即使以目前的能力，业界对其潜力的挖掘也远未达到10%。与此同时，有太多想法值得尝试，从概念上讲，这个领域感觉完全开放。正如我今年早些时候在我的Dwarkesh播客中提到的，我同时（表面上看似矛盾地）相信，我们将看到快速而持续的进展，并且仍有大量工作要做。系好安全带。
