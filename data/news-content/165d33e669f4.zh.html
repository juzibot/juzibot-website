在用 Rust 重写 Bun 这篇文章中，Jarred Sumner 提出了以下说法：

Claude Code v2.1.181（6 月 17 日发布）及之后的版本使用了 Bun 的 Rust 移植版。在 Linux 上启动速度提升了 10%，但除此之外，几乎没人注意到变化。平淡无奇才是好事。

我决定检查一下自己安装的 Claude Code，看看能否找到它正在使用 Rust 版 Bun 的证据。

我发现以下两条命令很有说服力：

strings ~/.local/bin/claude | grep -m1 'Bun v1'

对我来说，这输出了 Bun v1.4.0 (macOS arm64)。目前 GitHub 上 Bun 的最新版本是 5 月 12 日发布的 v1.3.14，因此 Claude 中出现的 v1.4.0 版本号支持了他们可能集成了一个尚未正式发布的 Bun 预览版的说法。

（更新：Rust 版本已经作为 Bun canary 发布了 - 运行 bun upgrade --canary 将会安装这个版本。）

strings ~/.local/bin/claude | grep -Eo 'src/[[:alnum:]_./-]+\.rs'

这输出了一个包含 563 个文件名的列表，开头是这些：

src/runtime/bake/dev_server/mod.rs
src/runtime/bake/production.rs
src/bundler/bundle_v2.rs

看起来 Rust 版的 Bun 确实正在数百万台不同的设备上投入生产使用。正如 Jarred 所说，“平淡无奇才是好事”。

更新：这里有一个来自 Ajan Raj 的巧妙技巧：

cat > /tmp/bun-version.ts <<'EOF'
console.log("embedded bun:", Bun.version);
process.exit(0);
EOF
BUN_OPTIONS="--preload=/tmp/bun-version.ts" claude --version

对我来说，这输出了 1.4.0。

这是5月17日的提交，它将 package.json 中的版本更新为 1.4.0。该版本此后一直未变，但也尚未作为正式标签发布，目前仅存在于 canary 版本中。
