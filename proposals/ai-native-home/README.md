# AI-native 首页重构提案 · ai-native-home

> 这是对官网首页的一版**重构原型提案**,以独立目录形式提交,**不改动现有 `index.html` 及主站构建体系**,便于预览、对比与评审。

## 预览

- 单文件,零构建依赖,直接用浏览器打开 `index.html` 即可。
- 唯一外部依赖:Google Fonts(Space Grotesk / Inter / JetBrains Mono)。

## 设计主张

1. **把 AI 当员工,不当功能列表**——用「我们的团队」替代产品卡片矩阵。
2. **结果驱动**——首页直接摆硬数据(转人工率 27%→2.73%、人效 3×、单线索成本砍 50–62% 等)。
3. **页面本身就是一次 AI-native 交互**——Hero 可输入对话、员工详情弹窗、工作台打字机、组织编排光点流动、买工具 vs 买结果计算器、实时部署地图、⌘K 命令面板。
4. **品类定义**:`Juzibot ≠ Chatbot,Juzibot = AI 员工系统`,用 demo 行为正面证明。
5. **视觉**:克制中性色 + 单一强调色 + 等宽字体点缀,工程文档式高信息密度。

## 技术要点

- 所有展示数据集中在顶部 `CONFIG` 对象,便于一键替换为真实数据或接接口。
- 动效只动 `transform`/`opacity`,全局尊重 `prefers-reduced-motion`。
- 交互元素带 `role`/`tabindex`/键盘支持,SVG 带 `aria-label`。
- 真实经纬度投影的中国部署地图。

## 已知待办(评审时请留意)

- **图标**:当前员工头像/节点用 emoji 占位,正式版应换成统一图标体系或真实产品截图。
- **配色**:近黑底 + 荧光绿为占位风格,可再议。
- **SEO/GEO**:内容由 JS 注入,尚未加 `<noscript>` 镜像 / JSON-LD / OG meta,作为后续 PR 补齐。
- **地图合规**:仅手动补了台湾岛,正式上线需按规范补南海诸岛 + 九段线标准小框。
- 尚未做浏览器自动化自测,建议评审者实开一遍。

## 数据口径

所有展示数据均为**占位示意**,表述参考 juzibot.com,请以官网最新口径为准。
